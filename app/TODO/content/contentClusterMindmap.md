## 📋 LINKING RULES MATRIX

| Link Type | Minimum Links | Maximum Links | Anchor Text Style |
|-----------|---------------|---------------|-------------------|
| **Homepage → Pillar** | 1 per pillar | 1 per pillar | "Explore [Destination]" |
| **Pillar → Cluster** | All clusters | All clusters | "Read our guide to [topic]" |
| **Cluster → Pillar** | 2-3 times | 2-3 times | "[Destination] Travel Guide" |
| **Cluster → Related Cluster** | 3-5 links | 8 links max | Descriptive phrase |
| **Cross-Pillar Links** | 2-3 links | 5 links max | Natural anchor text |
| **Blog → Cluster/Pillar** | 2-4 links | 6 links max | Contextual |

## 🎯 LINKING CHECKLIST FOR EACH ARTICLE

When you publish ANY new article, follow this checklist:

### **For Cluster Pages:**
- [ ] Link to pillar page in introduction (first 200 words)
- [ ] Link to pillar page in conclusion
- [ ] Link to 3-5 related cluster pages
- [ ] Include "Related Articles" section at bottom
- [ ] Add article to pillar page's Table of Contents
- [ ] Update pillar page with link to this cluster

### **For Pillar Pages:**
- [ ] Link to ALL cluster pages (15-20 links)
- [ ] Include Table of Contents with anchor links
- [ ] Link to other relevant pillars (2-3 cross-links)
- [ ] Link from homepage/navigation
- [ ] Add "Related Destinations" section

### **For Blog Posts (Non-Cluster):**
- [ ] Link to 2-3 relevant pillar pages
- [ ] Link to 2-3 relevant cluster pages
- [ ] Place links naturally in context
- [ ] Use descriptive anchor text

## 💡 PRO TIPS FOR YOUR SITE

1. **Use Your Existing Content First**
   - Your Mexico resort article = immediate cluster page
   - Start Mexico cluster around it
   - Leverage what you have!

2. **Create a Spreadsheet**
   - Track all clusters, pillars, links
   - Column 1: Article title
   - Column 2: URL
   - Column 3: Pillar it belongs to
   - Column 4: Links to (list URLs)
   - Column 5: Status (published/draft)

3. **Batch Your Writing**
   - Write all Mexico clusters in one month
   - Easier to maintain context and consistency
   - Faster internal linking setup

4. **Update Pillar Pages Monthly**
   - Add new clusters as you publish
   - Keep Table of Contents current
   - Refresh stats and recommendations

5. **Build Backlinks to Pillars**
   - Guest post and link to pillar pages
   - Pillars = link magnets
   - Cluster pages inherit authority
